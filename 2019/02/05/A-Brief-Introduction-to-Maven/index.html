A Brief Introduction to Maven

Posted on 2019-02-05   |   In Developmemt   |  

I have been using Java for coding for over three years, but I just started exploring Maven from last week. Although I knew Maven would be useful, I usually ignored it because I thought using Maven would add up the complexity and I need to learn how to use it first. But actually, it is not as hard as I thought. So I am writing this blog to share the usage of Maven to whom may have similar thought as I was.

Introduction

Apache Maven is a software project management and comprehension tool. Based on the concept of a project object model (POM), Maven can manage a project’s build, reporting and documentation from a central piece of information.

Maven’s primary goal is to allow a developer to comprehend the complete state of a development effort in the shortest period of time. In order to attain this goal there are several areas of concern that Maven attempts to deal with:

  • Making the build process easy
  • Providing a uniform build system
  • Providing quality project information
  • Providing guidelines for best practices development
  • Allowing transparent migration to new features

Actually, the real purpose of using Maven for the first time is most likely that Maven can help the developer to specify the project’s dependencies, instead of importing JAR files manually.

The main file we need to use for Maven is the pom.xml file. We can add dependencies, plugins or other settings in it. If you are familiar with Node.js or Django, it is very similar to package.json or requirements.txt. Next, we will download and explore the basic usage of Maven together.
